## Overview

This page documents all TypeScript types and interfaces exported by `safemocker`.

## SafeActionResult

The core result structure that matches `next-safe-action`'s return type:

<Badge>Core Type</Badge>

<AutoTypeTable path="src/types.ts" name="SafeActionResultType" />

<Callout type="info" title="Generic Type">
  The actual `SafeActionResult` type is generic:
  ```typescript
  interface SafeActionResult<TData>
  ```
  
  The interface above uses a concrete data type (`{ id: string; name: string }`) to demonstrate the structure. In practice, `TData` can be any type including primitives, objects, arrays, or complex nested structures.
  
  **Mutual Exclusivity**: Only one of `data`, `fieldErrors`, or `serverError` will be defined in a result. If validation fails, `fieldErrors` is set. If the handler throws, `serverError` is set. If everything succeeds, `data` is set.
</Callout>
